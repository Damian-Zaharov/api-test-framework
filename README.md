# API Automation Testing Framework (DummyJSON)

Профессиональный фреймворк для автоматизации тестирования [DummyJSON API](https://dummyjson.com). Проект демонстрирует
навыки построения масштабируемой архитектуры, валидации данных и построения наглядной отчетности.

## Стек технологий

* **Python 3.11+**
* **Pytest** — тестовый движок.
* **HTTPX** — современный асинхронный HTTP-клиент.
* **Pydantic V2** — строгая валидация схем данных.
* **Allure Reports** — интерактивные отчеты с визуализацией шагов.
* **Python-dotenv** — управление конфигурацией через переменные окружения.

## Архитектура проекта

Проект реализован с разделением ответственности (Layered Architecture):

* `core/` — "двигатель" проекта: базовый HTTP-клиент с логированием и обработкой ошибок.
* `clients/` — низкоуровневые API-клиенты (описание эндпоинтов).
* `services/` — слой бизнес-логики: сборка сценариев, управление токенами и маппинг в модели.
* `schemas/` — Pydantic-модели для автоматической валидации JSON-ответов.
* `utils/` — вспомогательные инструменты: кастомные валидаторы, декораторы ретраев (Retry) и логгеры.
* `tests/` — тестовые сценарии, разделенные по функциональным модулям.

## Ключевые особенности

* **Retry Mechanism**: Автоматический перезапуск тестов при нестабильности API (Flaky tests).
* **Deep Validation**: Валидация не только статус-кодов, но и структуры данных, типов полей и времени ответа.
* **Session Optimization**: Использование фикстур с областью видимости `session` для минимизации повторных авторизаций.
* **Rich Reporting**: Детальные Allure-отчеты с прикрепленными логами запросов/ответов и описанием шагов на языке
  бизнеса.

## Запуск проекта

1. **Клонирование репозитория:**
   ```bash
   git clone https://github.com
   cd api-test-framework
   ```

2. **Настройка окружения:**
   ```bash
   python -m venv .venv
   source .venv/bin/activate  # Для Windows: .venv\Scripts\activate
   pip install -r requirements.txt
   ```

3. **Запуск тестов и генерация отчета:**
   ```bash
   pytest --alluredir=allure-results
   allure serve allure-results
   ```
